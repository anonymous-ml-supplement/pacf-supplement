# Reproducibility Guide

This guide describes how to inspect the logged result summaries and how to rerun the main PACF experiments. The package uses public datasets and public pretrained backbones. Users must obtain each dataset and model from its official source and comply with the corresponding license and access requirements.

## Environments

Use separate environments when needed:

```bash
pip install -r environment/requirements_glue_clip.txt
pip install -r environment/requirements_llm.txt
pip install -r environment/requirements_sdxl.txt
```

The top-level `requirements.txt` is a compact generic file. Domain-specific files are more useful for full reproduction.

## Config-driven execution

Each YAML file has the form:

```yaml
domain: glue
entry: src/glue/pacf_glue.py
args:
  mode: paper
  task: sst2
```

The runner converts YAML arguments into command-line flags:

```bash
python3 scripts/run_config.py --config configs/glue/glue_sst2_r16_pacfkl_paper.yaml --dry_run
```

Remove `--dry_run` to launch the run.

## GLUE and CLIP

GLUE and CLIP configs instantiate the PACF-KL regime used for the no-extra-forward controlled experiments. Example commands:

```bash
python3 scripts/run_config.py --config configs/glue/glue_sst2_r16_pacfkl_paper.yaml
python3 scripts/run_config.py --config configs/clip/clip_cifar10_r16_pacfkl_paper.yaml
```

## LLM experiments

LLM configs instantiate PACF-Cons with one perturbed LoRA-subspace forward pass. Example command:

```bash
python3 scripts/run_config.py --config configs/llm/llm_chat_r8_pacfcons_paper.yaml
```

LLM experiments require access to the corresponding pretrained backbone and may require substantial GPU memory.

## SDXL qualitative experiment

The SDXL script supports qualitative LoRA, Flat-LoRA, and PACF-Cons personalization runs. Place instance images in `data/example_instance_images/` or pass a different relative path:

```bash
python3 scripts/run_config.py --config configs/sdxl/sdxl_3dicons_pacfcons_paper.yaml
```

The SDXL artifact is qualitative unless quantitative metrics are provided separately.

## Table inspection

Sanitized CSV exports and clean paper-table summaries are under `tables/`. The package does not silently modify numeric results. Table 5, Table 6, and Table 8 are included as paper-reported summaries because separate raw ablation, probe, or 10-seed logs were not present in the uploaded artifact.

## Randomness and seeds

Configs use the seed lists shown in the YAML files. Results may vary across hardware, library versions, and nondeterministic kernels. The paper reports seed means and standard deviations where available.

## Compute notes

PACF-KL has no additional perturbed forward pass. PACF-Cons uses one additional perturbed forward pass by default. Full LLM and SDXL reproduction may require high-memory GPUs. The package does not include institution-specific cluster commands.

## External judge models

Optional MT-Bench-style judging requires a user-provided judge model and API credentials at runtime. No credentials are included. Optional experiment logging is disabled by default. Scores can change with judge model calibration, so judge model names should be reported with any regenerated results.

## Known limitations

Some targeted ablation records from the paper were not present as separate raw logs in the uploaded supplemental ZIP. They are therefore not regenerated by this cleaned artifact unless additional evidence files are supplied.
